Git & GitHub



GitHub

GitHub est un service web d'hébergement et de gestion de développement de logiciels, utilisant le logiciel de gestion de versions Git. Ce site est développé en Ruby on Rails et Erlang par Chris Wanstrath, PJ Hyett et Tom Preston-Werner. GitHub propose des comptes professionnels payants, ainsi que des comptes gratuits pour les projets de logiciels libres. Le site assure également un contrôle d'accès et des fonctionnalités destinées à la collaboration comme le suivi des bugs, les demandes de fonctionnalités, la gestion de tâches et un wiki pour chaque projet. Le site est devenu le plus important dépôt de code au monde, utilisé comme dépôt public de projets libres ou dépôt privé d'entreprises. En 2018, GitHub est acquis par Microsoft pour 7,5 milliards de dollars.


Git

Git est de loin le système de contrôle de version le plus largement utilisé aujourd'hui. Git est un projet open source avancé, qui est activement maintenu. À l'origine, il a été développé en 2005 par Linus Torvalds, le créateur bien connu du noyau du système d'exploitation Linux. De plus en plus de projets logiciels reposent sur Git pour le contrôle de version, y compris des projets commerciaux et en open source. Les développeurs qui travaillent avec Git sont bien représentés dans le pool de talents disponible, et la solution fonctionne bien sur une vaste gamme de systèmes d'exploitation et d'environnements de développement intégrés (IDE).



Méthode d'utilisation



Utilisation locale (hors ligne) avec Git

Pour utiliser Git en local, il faut télécharger le logiciel Git sur le site officiel. Ensuite, voici certaines commandes bien pratiques pour son utilisation :

  • Configurer son nom :

    git config --global user.name ["Bob Smith"]


  • Configurer son email :

    git config --global user.email ["bob-smith@fakemail.com"]


  • Afficher le répertoir actuel :

    pwd


  • Lister des éléments du répertoire actuel :

    ls (pour list)


  • Effacer l'affichage de l'écran :

    clear


  • Ce placer dans le répertoire (dossier) voulu :

    CLIC DROIT + "GitBash Here"


  • Reculer d'un répertoire :

    cd .. (pour change directory)


  • Créer un dossier :

    mkdir [nom_dossier] / ["Nom du dossier"] (pour make directory)


  • Créer un fichier :

    touch [nom_du_fichier].[extension] / ["nom du fichier.[extension]"]


  • Initialiser le dossier actuel :

    git init


  • Observer le statut :

    git status


  • Ajouter un/des fichier.s :

    git add [nom_du_fichier.[extension]] / ["nom du fichier.[extension]"]


  • Commiter un fichier :

    git commit -m ["message"]


  • Afficher la liste des commits (du + récent au + ancien):

    git log


  • Afficher les différences entre le fichier et le commit :

    git diff [nom_du_fichier]


  • Créer une branche :

    git branch [nom_de_la_branche]


  • Afficher la liste des branches existantes (affichera aussi la branche actuelle) :

    git branch


  • Se positionner sur une branche :

    git checkout [nom_de_la_branche]


  • Supprimer une branche :

    git branch -d [nom_de_la_branche]


  • Forcer la suppression d'une branche (en cas de non vergeance de la branche) :

    git branch -D [nom_de_la_branche]


  • Afficher les différences entre les branches :

    git diff


  • Merger les modifications d'une branche depuis le "master" (bien être dans le master) :

    git merge [nom_de_la_branche]



Utilisation en ligne : remote server avec GitHub


  1. Vous voulez envoyer votre travail sur un dépôt distant non créer. Il suffit d'aller sur GitHub et de créer un nouveau "repositery".

    • Lier le dépôt distant :

      git remote add origin [lien_du_dépôt_donné_par_Github]

    • Pousser (envoyer) les modifications au dépôt distant :

      git push [branche du dépôt souhaitée, très souvent "origin"] [la branche qu'on veut envoyer]

  2. Vous voulez récupérer le travail d'un dépôt distant déjà créé.

    • Tirer (récupérer) le travail :

      git pull origin [nom de la branche, généralement "master" ou "main"]

    • Cloner un dépôt distant :

      git clone [lien_du_dépôt_donné_par_Github]
